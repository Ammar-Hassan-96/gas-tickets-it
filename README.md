# GAS Internal Tickets — German Auto Service
## نظام التيكتات الداخلي · v2.5 · Inbound/Outbound Model

نظام كامل للطلبات الداخلية بين إدارات الشركة بنموذج هرمي آمن مع **شفافية ذكية على فريق العمل**.

---

## 🆕 الجديد في v2.5 — نموذج Inbound/Outbound

**المشكلة اللي اتحلت:** المدير كان مش شايف الطلبات اللي موظفيه بعتوها لإدارات تانية.

**الحل:** Tab منفصل **"📤 طلبات فريقي الصادرة"** — المدير/المشرف يشوف كل طلبات موظفيه الصادرة لإدارات أخرى، بصلاحية **عرض + تعليق فقط** (تحديث الحالة يبقى حصري لإدارة الاستقبال).

---

## 🎯 نموذج الرؤية

### 📥 Inbound (طلبات إدارتي)
- مدير/مشرف: **سيطرة كاملة** — تعيين، تحديث حالة، أرشفة، تعليق
- موظف في الإدارة: يشوف الطلبات **المفتوحة** (open) عشان يستلمها
- Tab: "📥 طلبات {اسم الإدارة}"

### 📤 Outbound (طلبات فريقي الصادرة)
- مدير/مشرف: **عرض + تعليق فقط**
- ما يقدرش يحدث الحالة (إدارة الاستقبال هي المسؤولة)
- Tab: "📤 طلبات فريقي الصادرة"
- Banner واضح في صفحة التفاصيل: "📤 طلب صادر من فريقك · وضع المتابعة"

### 👤 الطلبات الشخصية
- أي موظف/قائد: يشوف طلباته كاملة في "🎫 طلباتي"
- **شفافية كاملة**: يشوف كل التفاصيل والتعليقات والمعين عليه

## 🏗️ الأدوار والصلاحيات الكاملة

```
👑 Super Admin (ammar.admin)
   └── كل الصلاحيات · بدون قيود

🏢 Manager (مدير إدارة واحدة)
   ├── Inbound: سيطرة كاملة على طلبات إدارته
   ├── Outbound: عرض + تعليق على طلبات فريقه الصادرة
   ├── إدارة موظفي إدارته (حذف/تعيين كلمة سر)
   └── تقارير فريقه

⭐ Supervisor (مشرف إدارة)
   ├── Inbound: تعيين، تحديث، تعليق
   ├── Outbound: عرض + تعليق
   └── لا يقدر يحذف تيكتات ولا موظفين

👤 Employee (موظف)
   ├── طلباته الشخصية (في أي إدارة)
   ├── الطلبات المعينة عليه
   └── الطلبات المفتوحة في إدارته (عشان يستلمها)
```

## 🚀 خطوات التفعيل

### 1️⃣ Supabase (شغل الـ migrations بالترتيب)

```sql
1) supabase-migration-internal.sql     (لو مش شغلت)
2) supabase-migration-roles-v2.sql     (لو مش شغلت)
3) supabase-migration-v3-data-fixes.sql
```

الـ migration v3 بيصلح التلقائي:
- المسافات الزائدة في أسماء الإدارات
- ترقية `ammar.admin` لـ `super_admin`
- تقرير تشخيصي كامل في الآخر

### 2️⃣ Netlify (push → auto-deploy)

### 3️⃣ داخل التطبيق
- ادخل كـ `ammar.admin`
- صفحة **"🔑 الأدوار"** → حدد دور وإدارة كل موظف → **حفظ كل التغييرات**

### 4️⃣ اختبر بسرعة
1. سجل دخول كموظف في إدارة X
2. افتح طلب لإدارة Y (أي إدارة تانية)
3. سجل دخول كمدير إدارة X
4. هتلاقي الطلب في **"📤 طلبات فريقي الصادرة"** ✅
5. افتحه → هتشوف Banner "📤 طلب صادر من فريقك · وضع المتابعة"
6. ما تقدرش تحدث الحالة، بس تقدر تعلق

## 🎭 السيناريوهات العملية

### سيناريو 1: موظف الحسابات يطلب IT support
- **أحمد** (الحسابات) يفتح طلب → IT
- **مدير الحسابات** (بيشوي) يشوف الطلب في "📤 طلبات فريقي الصادرة" — يتابع بس
- **مدير IT** (فام) يشوف الطلب في "📥 طلبات IT" — يحلها
- الاتنين يقدروا يعلقوا على الطلب

### سيناريو 2: موظف يطلب تظلم من HR
- الموظف يفتح الطلب → الموارد البشرية
- **مديره المباشر** يشوف الطلب (شفافية إدارية) — لكن ما يقدرش يحدث الحالة
- **مدير HR** هو اللي يعالج الطلب فعلاً

### سيناريو 3: Super Admin يتابع النظام كله
- "📥 كل الطلبات" → كل طلبات النظام
- "📤 طلبات فريقي الصادرة" → مش مهم (super_admin ماعندوش إدارة)
- Dropdown فلتر بالإدارة متاح بس له

## 🛠️ Troubleshooting

### الطلبات مش ظاهرة لمدير الإدارة
- نفذ `supabase-migration-v3-data-fixes.sql` (يصلح الـ trim تلقائياً)
- تأكد إن `users.department` مطابق لـ `tickets.target_department` بالظبط
- Ctrl+Shift+R في المتصفح

### Tab "طلبات فريقي الصادرة" فاضي
- ده طبيعي لو موظفيك لسه ما بعتوش أي طلب لإدارة تانية
- اختبر: ادخل كموظف وابعت طلب لإدارة غير إدارتك

### المدير شايف Banner "ليس لديك صلاحية"
- شغل Ctrl+Shift+R (cache قديم)
- تأكد من v2.5+ في الـ deploy

## 📝 التفاصيل التقنية

### طبقة Perm (في `app.js`)

```js
Perm.sameDeptAs(t)       // الطلب موجه لإدارتي
Perm.ownerFromMyDept(t)  // مقدم الطلب من إدارتي
Perm.isOutbound(t)       // طلب outbound بالنسبة لي
Perm.canSeeTicket(t)     // رؤية
Perm.canActOnTicket(t)   // تحديث حالة (Inbound فقط للقيادات)
Perm.canCommentOnTicket(t) // تعليق (Inbound + Outbound للقيادات)
Perm.canAssignTicket(t)  // تعيين
Perm.canDeleteTicket(t)  // حذف
```

### Helpers للفلترة

```js
visibleTickets()   // كل اللي المستخدم شايفه
inboundTickets()   // الموجه لإدارته
outboundTickets()  // صادر من فريقه لإدارات أخرى
```
